Páginas Dinámicas

Antes de comenzar la explicación es importantísimo recordar que Astro, mapea cada archivo de una Colección de Datos basándose en su nombre e ignorando su extensión. Astro llama a cada nombre de archivo slug.

En lugar de dedicarnos a enlazar archivo a archivo de nuestra colección, creamos un archivo de página dinámica al cual llamaremos [slug].astro y que nos permitirá renderizar cada archivo de una colección. Atendiendo a esta filosofía, para poder acceder a cada uno de los post, se crea un archivo con el nombre [page].astro para hacer la portada del blog/tienda.

La estructura de las páginas con dos colecciones (posts y productos) para Blog y tienda, quedaría así:


src/
└── pages/
    ├── blog/
    │    ├── [page].astro   <-- Portada de Blog/Índice de páginas (blog/1, blog/2...)
    │    └── [slug].astro   <-- Post individual
    ├── about.astro         <-- ¿Quiénes somos?
    ├── contacto.astro      <-- Página de contacto
    ├── index.astro         <-- La página de inicio de la web
    ├── tienda/
    │    ├── [page].astro   <-- Portada de la tienda (Galería de productos)
    │    └── [slug].astro   <-- Productos individuales

Archivo de página dinámica para renderizar elementos de la colección

Para renderizar el archivo sólo se necesita un archivo de página dinámica como éste:


---

import { getCollection } from 'astro:content';
import { Image } from 'astro:assets'; // Usamos el componente optimizado de Astro
import DefaultLayout from '../../layouts/DefaultLayout.astro';

// 1. Generamos las rutas para cada post de la colección
export async function getStaticPaths() {
  const posts = await getCollection('posts');
  return posts.map(post => ({
    params: { slug: post.slug },
    props: { post },
  }));
}

// 2. Recuperamos el post específico para esta página
const { post } = Astro.props;

// 3. Obtenemos el contenido del post (el cuerpo del Markdown/HTML)
const { Content } = await post.render();
---

<!-- 4. Maquetamos el diseño del post / producto -->
<DefaultLayout title={post.data.title}>
  <h1 class="text-3xl font-bold my-4">{post.data.title}</h1>
  <p>{post.data.description}</p>
  
  <figure>
    <!-- Usamos la herramienta de imágenes optimizadas de Astro -->
    <Image 
      src={post.data.image.url} 
      alt={post.data.image.alt}
      class="postImg"
      width={800} 
      height={450}
    />
  </figure>
  <hr>
  <!-- El componente <Content /> inyecta todo el cuerpo del .mdx o .md aquí -->
  <article class="prose">
    <Content />
  </article>

  <p><a href="/blog">Entradas del blog</a></p>
</DefaultLayout>

Donde:

  1. post.data guarda todas las propiedades del Frontmatter de cada archivo (en el caso del ejemplo, los posts) validadas por Zod.
  2. <Content/> en lugar de<slot />: En lugar de usar un <slot /> genérico de HTML (como en los layouts), se ejecuta una función await post.render() en el servidor para obtener el componente especial <Content/>. Al renderizarlo, Astro inyecta ahí todo el contenido Markdown o HTML del archivo.
  3. La estructura/estilo del archivo, se toman del Layout importado.

Si en el futuro se decide cambiar el diseño de todos los archivos de una colección, se haces en un solo archivo de Astro: ([slug].astro) sin tocar un solo Markdown/HTML de la misma.

Archivo de página dinámica para la paginación (Portada de la Colección)

Para crear un índice con páginas numeradas usaremos un archivo dinámico que se va a llamar [page].astro. Al llamar al archivo [page].astro (con la palabra reservada page entre corchetes), Astro entiende que va a dividir tu lista de posts en páginas.

El código de este archivo que genere el índice automáticamente, es el siguiente:


---
// src/pages/blog/[page].astro
import { getCollection } from 'astro:content';
import DefaultLayout from '../../layouts/DefaultLayout.astro';

// 1. Esta función le dice a Astro cómo dividir los posts
export async function getStaticPaths({ paginate }) {
  const todosLosPosts = await getCollection('posts');
  
  // Ordenar los posts por fecha (del más nuevo al más viejo)
  // Esta función cambia según el propósito de la colección
  const postsOrdenados = todosLosPosts.sort(
    (a, b) => b.data.pubDate.valueOf() - a.data.pubDate.valueOf()
  );

  // Paginar: Queremos mostrar, por ejemplo, 6 posts por página
  // El valor de pageSize lo indica.
  return paginate(postsOrdenados, { pageSize: 6 });
}

// 2. Astro pasa automáticamente la información de la página en la prop "page"
const { page } = Astro.props;
---

<DefaultLayout title={`Blog - Página ${page.currentPage}`}>
  <h1>Blog sobre Gatitos</h1>

  <!-- 3. Pintamos los posts de la página actual -->
  <div class="posts-grid">
    {page.data.map((post) => (
      <article class="post-card">
        <h2>{post.data.title}</h2>
        <p>{post.data.description}</p>
        <a href={`/blog/${post.slug}`}>Leer más</a>
      </article>
    ))}
  </div>

  <!-- 4. Botones de Navegación del Índice -->
  <nav class="paginacion">
    {page.url.prev ? <a href={page.url.prev}>← Anterior</a> : ← Anterior</span>}
    
    <span class="pagina-actual">Página {page.currentPage} de {page.lastPage}</span>
    
    {page.url.next ? <a href={page.url.next}>Siguiente →</a> : <span class="disabled">Siguiente →</span>}
  </nav>
</DefaultLayout>

<style>
  .posts-grid { display: grid; grid-template-columns: repeat(auto-fill, minmax(300px, 1fr)); gap: 2rem; margin-bottom: 2rem; }
  .post-card { border: 1px solid #eee; padding: 1.5rem; border-radius: 8px; }
  .paginacion { display: flex; justify-content: space-between; align-items: center; margin-top: 3rem; }
  .disabled { color: #ccc; cursor: not-allowed; }
</style>

pageSize es el límite de tarjetas (cards) de posts que se mostrarán en cada página del índice.

Si en una colección se tienen por ejemplo, 15 posts en total y se configura pageSize: 6, Astro generará 3 Urls de forma automática:

  1. Página 1 (/blog/1): Mostrará los posts del 1 al 6 (6 cards).
  2. Página 2 (/blog/2): Mostrará los posts del 7 al 12 (6 cards).
  3. Página 3 (/blog/3): Mostrará los posts restantes del 13 al 15 (3 cards).

Astro calcula todo esto de manera automática. Solo con cambiar el número de pageSize a 9, 12, etc..., se reestructurarán todas las páginas y los enlaces de Anterior y Siguiente al compilar la web.
