Content Collections para crear colecciones de archivos

Las Content Collections (Colecciones de Contenido) son una característica nativa que ya viene integrada en el núcleo de Astro (desde la versión 2.0).

Lo único que debemos hacer para empezar a usar las Colecciones de Contenido es estructurar las carpetas de una forma específica y crear un archivo de configuración.

Paso 1: Crear una carpeta llamada Content dentro de src

Astro reserva una carpeta especial llamada src/content/ para las colecciones. Cuando querramos que Astro genere automáticamente las entradas de un blog, los productos de una tienda, etc... sólo debemos guardarlos en subcarpetas dentro de la carpeta content (el nombre de la subcarpeta será el nombre de tu colección).

Por ejemplo, para que Astro pueda leer automáticamente todas las entradas de un blog, podemos crear una subcarpeta llamada entradas o posts. La ruta completa a la colección sería por ejemplo, src/content/posts.

Paso 2: Crear un archivo llamado config.ts en la carpeta src/content/


import { defineCollection, z } from 'astro:content';

const postsCollection = defineCollection({
  type: 'content',
  schema: ({ image }) => z.object({  // valida las imágenes del schema
    title: z.string(),
    pubDate: z.date(),
    description: z.string(),
    author: z.string(),
    // Para indicar una imagen
    image: z.object({
      url: image(), // <--- Astro valida que el archivo exista de verdad
      alt: z.string(),
    }),
    tags: z.array(z.string()),
  }),
});

// Exportas tus colecciones para que Astro las reconozca
export const collections = {
  'posts': postsCollection,
};

Imaginemos ahora que tenemos dos colecciones: una colección de posts para un blog y otra de productos para una tienda. ¿Cómo quedaría el config.ts?


import { defineCollection, z } from 'astro:content';

// 1. DEFINIR LA COLECCIÓN DE POSTS
const postsCollection = defineCollection({
  type: 'content', // 'content' para archivos Markdown/MDX
  schema: ({ image }) => z.object({
    title: z.string(),
    pubDate: z.date(),
    description: z.string(),
    author: z.string(),
    image: z.object({
      url: image(), // Valida que la imagen de portada exista
      alt: z.string(),
    }),
    tags: z.array(z.string()),
  }),
});

// 2. DEFINIR LA COLECCIÓN DE PRODUCTOS
const productosCollection = defineCollection({
  type: 'content', // O puedes usar 'data' si prefieres archivos .json o .yaml en lugar de Markdown
  schema: ({ image }) => z.object({
    name: z.string(),
    price: z.number(),
    stock: z.number(),
    category: z.enum(['comida', 'juguetes', 'accesorios']), // Restringe a estas categorías
    isFeatured: z.boolean().default(false),
    productImage: image(), // Aquí también puedes usar el validador de imágenes de Astro
  }),
});

// 3. EXPORTAR TODAS LAS COLECCIONES
// Las claves de este objeto (posts, productos) deben coincidir EXACTAMENTE con los nombres de tus carpetas

export const collections = {
  'posts': postsCollection,
  'productos': productosCollection,
};

Tipos de Colecciones

Las Content Collections de Astro están diseñadas nativamente para trabajar con dos tipos de formatos (type: 'content' y type: 'data'):

  1. Archivos de contenido type: 'content': Archivos Markdown (.md) y MDX (.mdx).
  2. Archivos de datos estructurados type: 'data': Archivos JSON (.json) y YAML (.yaml).

Si tienes código HTML que necesitas reutilizar o meter dentro de tus colecciones, usa archivos MDX.

Si el contenido tiene que estar sí o sí dentro de una Content Collection para beneficiarse de la validación de Zod y la facilidad de usar getCollection y ya se tiene hecho en .html o el diseño requiere algo más de lo que ofrece MarkDown, la extensión .mdx es la mejor aliada

Al cambiar la extensión de los archivos de .html a .mdx, se puedes escribir HTML puro y duro dentro de ellos, en vez de sólo Markdown. MDX es una extensión que permite mezclar Markdown, componentes de Astro y HTML estándar en un mismo archivo.

Para que Astro entienda los archivos .mdx en un proyecto, se necesita añadir la integración oficial


pnpm astro add mdx
