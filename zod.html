Validar con Zod

Zod es una librería de validación de esquemas integrada en Astro, que funciona de la mano con TypeScript. Se usa para asegurar que el Frontmatter de tus archivos de colección de contenidos, tenga exactamente la estructura y los tipos de datos que se espera.

Los tipos de campo que se usan en el config.ts de Astro con Zod, organizados por categorías pueden ser:

1. Tipos de Datos Primitivos

Son los bloques de construcción básicos para cualquier Frontmatter:

Tipo en Zod Tipo en JS/TS Ejemplo en Frontmatter Descripción
z.string() string title: 'El Gato Bombay' Texto plano
z.number() number price: 19.99 Números enteros o decimales.
z.boolean() boolean isFeatured: true Valores verdadero/falso
z.date() Date pubDate: 2026-07-14 Fechas. Astro las convierte automáticamente a objetos `Date` de JS.

2. Estructuras de Datos Complejas

Guardar listas, objetos o estructuras anidadas:

Arreglos. Se debe indicar el tipo de datos del arreglo: z.array(z.string())

En el Frontmatter:


# Frontmatter
tags: ["astro", "gatos", "blog"]

Objetos anidados. Para agrupar propiedades relacionadas: z.object({ ... })


// Zod
author: z.object({
  name: z.string(),
  twitter: z.string()
})

En el Frontmatter


# Frontmatter
author:
  name: 'Purina'
  twitter: '@purina_es'

Imágenes: image()

Astro inyecta una función especial en el esquema para resolver rutas de imágenes físicas:


// Zod (en el config.ts de Astro)
schema: ({ image }) => z.object({ cover: image(), }) // Valida que la imagen exista en la ruta indicada

3. Modificadores Prácticos

Zod permite flexibilizar las reglas para que no todo sea estrictamente obligatorio:

Campos opcionales .optional():

El campo puede no estar en el archivo y no dará error (su valor será undefined).


subtitle: z.string().optional()

Valores por defecto .default(valor):

Si el usuario no lo escribe en el Frontmatter, Astro le asignará automáticamente el valor que definas.


draft: z.boolean().default(false)

Permitir valores nulos .nullable():

Permite que un valor sea explícitamente null.


discountPrice: z.number().nullable()

4. Validaciones y Tipos Avanzados

Para hacer que los datos sean aún más robustos:

Enumeraciones z.enum([...]):

Obliga que un valor sea únicamente uno de los elementos de la lista.


category: z.enum(['comida', 'juguetes', 'accesorios'])

Validaciones de Texto:

Se puede exigir formatos específicos para los strings, como correos electrónicos o URLs:


email: z.string().email()      // Valida formato de correo
link: z.string().url()         // Valida formato de URL (http/https)

Validaciones Numéricas:


price: z.number().positive()   // Obliga a que sea mayor a 0
age: z.number().int()          // Obliga a que sea un número entero
