Content Collections en Markdown

Una vez se crea el archivo src/content/config.ts podemos comenzar a crear los posts.

El Frontmatter de cada post, deberá contener los metadatos ajustados al archivo config.ts. Es decir, para el siguiente archivo, src/content/posts/post1.md...


---
# Frontmatter
title: 'El Gato Bombay'
pubDate: 2022-07-01
description: 'Parece una pantera en miniatura. Es un gato elegante, pero fuerte y tiene una naturaleza curiosa, activa y muy afectuosa.'
author: 'Purina'
image:
  url: '../../assets/gato-bombay.webp'
  alt: 'Gato Bombay'
tags: ["astro", "bloguear", "gatos", "bombay"]
---

CODIGO md ó mdx DEL POST

Se deberá tener el siguiente archivo src/content/config.ts:


import { defineCollection, z } from 'astro:content';

const postsCollection = defineCollection({
  type: 'content',
  schema: ({ image }) => z.object({  // valida las imágenes del schema
    title: z.string(),
    pubDate: z.date(),
    description: z.string(),
    author: z.string(),
    // Para indicar una imagen
    image: z.object({
      url: image(), // <--- Astro valida que el archivo exista de verdad
      alt: z.string(),
    }),
    tags: z.array(z.string()),
  }),
});

// Exportar la colección
export const collections = {
  'posts': postsCollection,
};

Dentro de la carpeta post, se pueden mezclar perfectamente archivos .md y .mdx, aunque se deberá tener en cuenta lo siguiente: Astro, mapea el id (al cual llama slug) de cada post basándose en su nombre del archivo e ignorando su extensión. Por tanto hay que tener cuidado de no tener dos archivos con el mismo nombre (ejemplo: mi-post.md y mi-post.mdx), para no causar un conflicto de nombres.

Recordar

Las imágenes de los archivos de una colección se guardan dentro de src/assets/ para que Astro las optimice al momento de renderizar.

Para que los archivos .mdx se rendericen correctamente, hay que asegúrarse de tener instalada e integrada la librería MDX en el proyecto Astro. Se puede agregar rápidamente ejecutando en la terminal:


  pnpm astro add mdx
